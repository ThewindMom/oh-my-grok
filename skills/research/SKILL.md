# Research

Find authoritative information and cite it precisely.

## Process

1. **Identify** what information is needed and why.
2. **Search** available sources:
   - Local files and installed documentation
   - Cached repositories
   - Official documentation (if web tools available)
   - Source code
3. **Cite** exact references:
   - URLs with full paths
   - File paths with line numbers
   - Version numbers and commit hashes
4. **Quote** relevant excerpts verbatim.
5. **Distinguish** authoritative sources from assumptions.

## Source hierarchy

1. Official documentation (highest authority)
2. Source code
3. Test files (show intended behavior)
4. Comments and READMEs
5. Community resources (lowest authority)

## Honesty

- If you can't find authoritative information, say so.
- Don't fabricate URLs, version numbers, or API signatures.
- If web research tools aren't available, work from local knowledge and state the limitation.
