# Review Work

Review changes with rigor. Don't rubber-stamp.

## Step 1: Inspect the diff

```
git diff --staged
git diff
```

Understand every changed line. Ask: does this change make sense?

## Step 2: Run tests

Run the full test suite for the affected packages. If any test fails, that's a blocker.

## Step 3: Check for regressions

- Are there other areas that depend on the changed code?
- Are there integration tests that cover the change?
- Are there edge cases that aren't tested?

## Step 4: Verify completeness

- Are all plan tasks addressed?
- Are there TODOs or placeholders left?
- Is documentation updated?
- Are error paths covered?

## Step 5: Check for AI slop

- Remove AI-generated praise comments ("Great function!")
- Remove comments that restate the code ("// This function returns X")
- Remove attribution comments ("// Generated by AI")
- Remove end-of-block comments ("// end of function")

## Output

Return exactly one of:
- **PASS** — with a brief summary of what was verified
- **BLOCKERS** — with a list of:
  - Blocker description
  - Evidence (file:line, test output)
  - Required fix
